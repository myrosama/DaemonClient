# Bug Fixes Deployed - 2026-04-26 19:25 UTC

## Critical Bugs Fixed

### 1. Video Thumbnails Not Generating ⚠️ CRITICAL
**Problem:** Videos uploaded from mobile or web had no thumbnails, showing broken image icons in the gallery.

**Root Cause:** 
- Thumbnail generation only worked for non-HEIC images (line 369)
- Used `sendDocument` which doesn't generate thumbnails for videos
- HEIC files were explicitly excluded from thumbnail generation

**Fix:**
- Changed to use `sendVideo` for video files → Telegram auto-generates thumbnails
- Extended thumbnail generation to handle both images AND videos
- Removed HEIC exclusion (Telegram-generated thumbs are always JPEG anyway)
- Added safety check: only attempt thumbnail generation for single-chunk, unencrypted files

**Files Changed:**
- `immich-api-shim/src/assets.ts` lines 368-401

**Code Changes:**
```typescript
// OLD: Only images, no HEIC, no videos
if (!telegramThumbId && mimeType.startsWith('image/') && !isHeic && file) {
  // sendPhoto only...
}

// NEW: Images AND videos, proper method selection
const isVideo = mimeType.startsWith('video/') || duration;
const isImage = mimeType.startsWith('image/');
const isSingleChunk = totalChunks === 1;

if (!telegramThumbId && (isImage || isVideo) && !isEncryptedByServer && isSingleChunk && file) {
  if (isVideo) {
    // Use sendVideo → gets thumb from video.thumb.file_id
    thumbForm.append('video', ...);
    // Telegram extracts thumbnail automatically
  } else {
    // Use sendPhoto → gets thumb from photo[0].file_id
    thumbForm.append('photo', ...);
  }
}
```

### 2. MIME Type Detection Broken for Mobile Uploads ⚠️ CRITICAL
**Problem:** Mobile uploads (especially iPhone) sent generic MIME types like `application/octet-stream` or incorrect `video/quicktime`, causing:
- Videos displayed as images
- Wrong file handling
- Broken thumbnail generation

**Root Cause:**
- Mobile apps don't always send correct MIME types
- No fallback to file extension detection

**Fix:**
- Added comprehensive extension-based MIME type detection
- Covers all common video formats (mp4, mov, avi, webm, mkv)
- Covers all common image formats (jpg, png, webp, heic)
- Only applies when MIME is missing or generic

**Files Changed:**
- `immich-api-shim/src/assets.ts` lines 287-301

**Code Changes:**
```typescript
// Fix MIME type detection based on file extension if missing or generic
if (!mimeType || mimeType === 'application/octet-stream' || mimeType === 'video/quicktime') {
  const ext = fileName.toLowerCase().split('.').pop();
  if (ext === 'mp4') mimeType = 'video/mp4';
  else if (ext === 'mov' || ext === 'qt') mimeType = 'video/quicktime';
  // ... all formats
}
```

### 3. Multi-Chunk Files Breaking Thumbnails
**Problem:** Large files split into multiple chunks had no thumbnails but attempted to serve them anyway, causing:
- Broken image icons
- 404 errors silently swallowed
- Gallery showing empty squares

**Root Cause:**
- No check for multi-chunk files when serving thumbnails
- Fallback logic attempted to stream original (which requires ReadableStream)

**Fix:**
- Added explicit check: multi-chunk files without thumbnails → return 404
- Better logging to debug missing thumbnails
- Proper fallback chain: thumb → original (single-chunk only)

**Files Changed:**
- `immich-api-shim/src/assets.ts` lines 476-494

**Code Changes:**
```typescript
// For multi-chunk files without thumbnails, we can't serve them as thumbnails

  return json({ message: 'Thumbnail not available for multi-chunk asset' }, 404);
}
```

## New Features Added

### 4. Search Functionality ✨ NEW
**Feature:** Full-text search by filename, date range, favorites, archive status, and asset type.

**Endpoint:** `POST /api/search/metadata`

**Request Body:**
```json
{
  "q": "vacation",                    // Filename search
  "takenAfter": "2024-01-01T00:00:00Z",  // Date range start
  "takenBefore": "2024-12-31T23:59:59Z", // Date range end
  "isFavorite": true,                 // Filter favorites
  "isArchived": false,                // Filter archived
  "type": "IMAGE"                     // Filter by type (IMAGE/VIDEO)
}
```

**Implementation:**
- In-memory filtering (no external search service)
- Still $0 cost
- Filters Firestore `photos` collection
- Returns standard asset response format

**Files Created:**
- `immich-api-shim/src/search.ts` (NEW)

**Files Changed:**
